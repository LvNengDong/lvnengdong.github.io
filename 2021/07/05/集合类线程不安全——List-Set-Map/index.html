Posted on 2021-07-05 In Collection

ArrayList 线程不安全

带着问题学习：

在 Java8 中，

  1. new ArrayList<>() 的底层数据结构是什么？
  2. new ArrayList<>() 的初始长度是多少？
  3. 数组长度一旦初始化就不能再修改了，Java 底层对数组进行扩容使用的方法是什么？
  4. 数组扩容的标准是什么？每次扩容多大？
  5. ArrayList 是线程安全的吗？
  6. 写一个 ArrayList 不安全的例子？
  7. 如何解决 ArrayList 线程不安全？有几种方法？

答案：

  1. new ArrayList<>() 底层默认的数据结构是 Object ，
  2. 初始长度为 10。

即 new ArrayList<>() 初始会在内存空间中创建一个 Object[10] 的数组对象。（不完全正确）

  1. Java 底层对数组进行扩容使用的是 Arrays.copyOf 方法，再向下一层是 System.arraycopy 方法（这是一个本地方法）。

    • ArrayList 是集合一个对象，底层使用数组实现的。对于数组而言，其容量在第一次分配内存空间时就已经确定好了，不能再发生改变了。扩容就是新创建一个更大容量的数组，将原先数组中的全部数据拷贝到新数组中去。

  2. 数组扩容的标准是正常情况下每次扩张一半，即每次扩张到原来的 1.5 倍。

    • 比如，新建一个 ArrayList 数组，数组容量最初为 10，第一次扩容后从 10 扩展到 15，第二次则是从 15 扩展到 22。
    • 扩展：HashMap 的初始长度是 16，每次扩容容量增加一倍。

    如果扩张到原来的 1.5 倍仍然不够用的话，那么就扩张到与有效元素个数相同的容量；

    扩张是有最大值的，最大值为 Integer.MAX_VALUE。

  3. ArrayList 是线程不安全的。

  4. 例子见下文

  5. 解决 ArrayList 的线程不安全有多种方法，根据实际需要自行选择。本文罗列出来了 X 种。

ArrayList 深度解析

ArrayList 实现了 List 接口，继承了 AbstractList 类，底层是数组实现的，一般我们把它认为是可以自动扩容的数组。

它是非线程安全的，一般多用于单线程环境下（与 Vector 最大的区别就是，Vector 是线程安全的，但是 ArrayList 性能更好）；

它实现了 Serializable 接口，支持序列化，能够通过序列化传输；

它实现了 RandomAccess 接口，支持快速随机访问，这里表现为可以通过下标直接访问（底层是数组实现的，所以可以直接用数组下标来索引）；

实现了 Cloneable 接口，能被克隆。

  • 数组是不可以自动扩容的！

带参构造器

public ArrayList(Collection<? extends E> c)

这个构造方法是直接将一个 Collection 集合中所有的元素拷贝到 ArrayList 中。

  1. 首先将 Collection c 转换成一个 Object[] 数组 ，elementData 就是转换后的数组；（数组的 length 属性指的是数组的容量而非数组中有效元素的个数）
  2. 先获取 elementData 的容量并将其赋值给 size，这里的 size 是一个实例变量，用于记录 elementData 的元素个数（而非数组的容量）。
    • 如果 size 不为 0 （Collection c 不为空）则调用 Arrays.copyOf() 方法拷贝 elementData 中的元素到新的内存空间中；
    • 否则（Collection c 为空）让 elementData 指向一个类初始化时就已经创建好的空数组对象。

ArrayList 的动态扩容

带着问题学习：

  1. 通过上面的学习，我们还是不知道为什么无参构造器默认指向了一个容量为 0 的空数组，而在注释中却说是一个初始容量为 10 的空数组呢？
  2. 通过上面的学习我们并没有看到 ArrayList 是动态扩容的，那么为什么说 ArrayList 可以看做是一个动态扩容的数组呢？

接下来进入 ensureCapacityInternal() 方法查看其具体实现：

  1. ensureCapacityInternal 方法

ensureCapacityInternal() 继续调用 ensureExplicitCapacity 和 calculateCapacity 方法保证数组容量的问题：

  1. calculateCapacity 方法

  • **minCapacity **是要容纳新添加的元素所需要的最小数组容量
  • DEFAULT_CAPACITY 是一个常量 10，表示默认数组容量
  • elementData 是全局的实例变量

通过这一步来判断，当前 elementData 是否为容量为零的数组（显式地指定初始化容量为 0 或者调用无参构造器），

  1. 如果是，则使用 Math.max(DEFAULT_CAPACITY, minCapacity) 选择两个参数中较大的。其中 DEFAULT_CAPACITY==10，minCapacity==size+1。可以看出，如果 elementData 为空的话，size+1 即 minCapacity 将会是 1，取较大值后返回的是 10。所以说无参构造器默认初始化一个容量为 10 的数组。但其实并不严格，我认为这里应该说：无参构造器默认不会初始化数组，而是指向一个在类加载阶段就已经创建好了的容量为 0 的空数组，只有再调用了 add() 方法时，才会在堆空间中初始化一个对象，这时候这个对象的容量才为 10。
  2. 得到了 minCapacity 数组容量这个值后，当前方法出栈继续调用上一个方法 ensureExplicitCapacity。
  1. ensureExplicitCapacity 方法

  1. modCount++ 是用于在快速失败机制中判断使用的，与本文相关性不大，暂时略过。

  2. 接下来判断，新增元素后数组中元素个数是否超过了当前数组的容量 elementData.length，如果超过，则调用 grow 方法进行扩容。

  1. grow 方法

  1. minCapacity 是新增元素后全部有效元素的个数（可以出现有效元素个数大于数组容量的情况）或 10
  2. oldCapacity 是原来的 ArrayList 集合的容量
  3. 由于在上一个方法中已经知道原来集合的容量放不下新增的元素了，所以这里首先要进行扩容。oldCapacity >> 1 是位运算的右移操作，右移一位相当于除以 2，所以这句代码就等于 int newCapacity = oldCapacity + oldCapacity/2，即容量扩大为原来的1.5倍
  4. 让扩容后得到新的集合 newCapacity 与 minCapacity 进行比较，如果仍然不够，则直接让 newCapacity == minCapacity，而不再是计算 1.5 倍的扩容。（即如果扩容 1.5 倍后仍然容量不够，那就直接让数组的容量等于元素的个数）
  5. 如果扩容后（第3或4步）发现新数组的容量超过了规定的最大容量 MAX_ARRAY_SIZE，则调用 hugeCapacity方法进行处理。
    • MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8，因为数组下标是整型的数据，所以其取值范围不能超过这个范围。

所有的处理结束后，调用 Arrays.copyOf 方法将旧数组中的所有元素拷贝到新数组中。

  1. hugeCapacity 方法

如果新增元素后需要的最小容量大于 MAX_ARRAY_SIZE ，则分配一块长度为 Integer 的最大值 Integer.MAX_VALUE 的内存用于数组的存储，如果小于等于，则分配一块长度为 MAX_ARRAY_SIZE 内存用于数组的存储。

方法执行完后，当前栈帧出栈，执行上一个方法。

  1. Arrays.copyOf
  1. 执行完 hugeCapacity 方法后， grow 方法继续被执行，调用 Arrays.copyof 方法，进行复制操作。

  1. Arrays.copyof 底层真正执行的是 System.arraycopy 方法。

  1. 而 System.arraycopy 方法则是一个本地方法。
