函数式接口

包含且仅包含一个抽象方法的接口，称为函数式接口。（不是规定接口中只能有一个方法）

@FunctionalInterface注解：我们可以通过在任意的函数式接口上使用@FunctionalInterface注解来检查本接口是否为函数式接口。同时，JavaDoc也会包含一条声明，说明这个接口是一个函数式接口。

Lambda表达式

所谓的 Lambda 表达式，就是对接口中抽象方法的实现。

Lambda是一个匿名函数，我们可以把Lambda表达式理解为一段可以传递的代码（将代码像数据一样进行传递）。可以写出更简洁、更灵活的代码。

Lambda表达式，也可以称为闭包，是Java8的新特性。

Lambda允许把函数作为一个方法的参数（函数作为参数传递进方法中）。

Lambda 表达式语法

格式：

Java8 中引入了一个新的操作符->，它将 Lambda 分为了两个部分：

  • 左侧：Lambda 表达式的参数列表；对应接口中抽象方法的参数列表；
  • 右侧：Lambda 表达式的方法体；对应接口中抽象方法的实现类的方法体。

Lambda 表达式对接口的要求

  • 需要函数式接口的支持。即@FunctionalInterface接口

  • 因为函数式接口中有且仅有一个抽象方法，所以使用 Lambda 表达式实现的就是该抽象方法。

Lambda表达式的特征：

  • 可选的类型声明：不需要声明参数类型，因为JVM编译器可通过上下文推断出数据类型。
  • 可选的参数圆括号：一个参数无需定义圆括号，但多个参数需要定义圆括号
  • 可选的大括号：如果主体只包含了一条语句，就不需要使用大括号。
  • 可选的返回关键字：如果主体只有一条语句且为返回值语句，则可以省略return关键字和大括号（return关键字和大括号要么全都省略，要么全都不省略）。

Lambda 语法优化

  1. 简化参数类型：可以不写参数类型，但是必须所有参数都不写参数类型
  2. 简化参数小括号，如果只有一个参数则可以省略参数小括号。
    • 有多个参数不能省略小括号；
    • 没有参数也不能省略小括号；
  3. 简化方法体大括号，如果方法体中只有一条语句，则可以省略方法体大括号
    • 如果方法体中唯一的一条语句是 return 语句，则进行省略时需要把方法体大括号和return关键字一起省略。

Lambda 表达式只是实现了方法，但是并没有调用方法。

方法引用

概念

在前面的学习中，Lambda 方法体中的方法实现都是我们自己写的。

有时候 Lambda 表达式方法体中的具体实现不是必须要我们自己重写，我们可以引用已被实现的方法作为 Lambda 表达式的方法体，这就叫做方法引用！

可以理解为方法引用是 Lambda 表达式的另外一种表现形式。

语法格式

  • 对象::实例方法名
  • 类名::静态方法名
  • 类名::实例方法名

Stream API

概念

Stream：流

Stream 是 Java8 中处理集合的关键抽象概念，它可以指定你希望对集合进行的操作，可以执行非常复杂的查找、过滤和映射数据等操作。使用 Stream API 对集合数据进行操作，就类似于使用 SQL 执行数据库查询。 也可以使用 Stream API 来并行执行操作。 简而言之，Stream API 提供了一种高效且易于 使用的处理数据的方式。

Stream 用于操作数据源（集合、数组等）所生成的元素序列。

包：java.util.stream.*

Tip：

  • Stream 自己不会存储元素。
  • Stream 不会改变源对象。相反，它们会返回一个持有结果的新的 Stream 对象。
  • Stream 操作是延迟 执行的。这意味着它们会等到需要结果的时候才执行。

Stream操作的三个步骤

  1. 创建Stream对象
    • 一个数据源（如：集合、数组）
    • 获取一个流对象
  2. 中间操作
    • 一个中间操作链，对包含数据源数据的流对象进行处理
  3. 终止操作（终端操作）
    • 一个终止操作 ，执行中间操作链，并产生结果。

创建Stream对象

创建Stream对象的4种方法

  1. Java8 中的 Collection 接口被扩展，提供了两个获取流的方法。
  2. 通过 Arrays 中的的静态方法 stream() 可以获取数组流。
  3. 通过 Stream 类中的静态方法 of() 获取流对象
  4. 通过 Stream 类中的静态方法iterate或generate创建无限流。
    • 无限流就是如果不加限制 ，将会一直生成数据的流。
类 方法 说明
Collection default Stream<E> stream() 返回一个顺序流
- default Stream<E> parallelStream() 返回一个并行流
Arrays static <T> Stream<T> stream(T[] array) 返回一个流
Stream<T> public static<T> Stream<T> of(T... values) 返回一个流
Stream<T> public static<T> Stream<T> iterate(final T seed, final UnaryOperator<T> f) 迭代

多个中间操作可以连起来形成一个流水线，除非流水线上触发终止操作，否则中间操作不会执行任何的处理！而在终止操作时一次性全部处理，称为“惰性求值”。

并行流与串行流

并行流就是把一个内容分成多个数据块，并用不同的线程分别处理每个数据块的流。

Java8 中将并行进行了优化，我们可以很容易的对数据进行并行操作。Stream API 可以声明性的通过parallel()与sequential()在并行流和串行流之间进行切换。
