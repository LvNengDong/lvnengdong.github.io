什么是AJAX请求

AJAX（Asynchronous JavaScript And XML）：异步的 JavaScript 和 XML

Ajax 在浏览器与 Web 服务器之间使用异步数据传输（HTTP 请求），这样就可使网页从服务器请求少量的信息，而不是整个页面。

AJAX是一种客户端通过JavaScript发起异步请求，局部更新页面的技术。

  1. AJAX请求是局部更新，浏览器地址栏不会发生变化；
  2. 局部更新不会舍弃原来页面的内容。

异步和同步

  1. 异步和同步（客户端和服务器端相互通信的基础上）：

    • 同步：客户端向服务器端发送请求，在服务器处理请求的时间段内，客户端必须等待服务器端做出响应后才能进行下一步操作。在等待的期间客户端不能进行其它的操作。
    • 异步：客户端向服务器端发送请求，在服务器处理请求的时间段内，客户端也要等待服务器端的响应，但在等待响应的期间，客户端可以进行其它的操作。

  • Ajax是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。
  • 通过在后台与服务器进行少量数据交换，Ajax可以使整个网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的部分进行更新。
  • 传统的网而言（不适用Ajax）如果需要更新内容，就必须重载整个网页页面。

Ajax请求的实现方式

AJAX请求定义在<script>标签内。所以可以通过JS或JS的框架发送异步请求。

JavaScript实现Ajax请求(了解)

发送异步请求的步骤：

  1. 创建XMLHttpRequest对象：xmlhttp

  2. 客户端向服务器发送请求：

    • 建立连接：xmlhttp.open()
    • 发送请求：xmlhttp.send()
    方法 描述
    规定请求的类型、URL以及是否异步处理请求。
    [method：GET/POST]
    open(method, url, async) [URL：文件在服务器上的位置]
    [async：true(异步)/false(同步)]
    send([string]) 将请求发送到服务器。
    string：仅用于POST请求

  3. 服务器接收请求执行业务的逻辑。

  4. 客户端接收并处理服务器的响应结果。

客户端的浏览器监听服务器中响应状态信息，当监听到响应成功后，获取服务器的响应结果。

  • XMLHttpServlet对象中有3个重要属性：

    ID 属性 描述
    1 onreadystatechange 事件，绑定函数，每当readyState属性改变时，就会调用该函数
    存放XMLHttpServlet的状态。从0到4发生变化
    0：请求未初始化
    2 readyState 1：服务器已建立连接
    2：请求已接收
    3：请求处理中
    4：请求已完成，且响应已就绪
    3 status 200：‘OK’
    404：未找到页面

    • 在 onreadystatechange 事件中，我们定义当服务器响应已做好被处理的准备

    • 并且 当 readyState 等于 4 且状态为 200 时，表示响应已就绪，这时才处理响应结果
