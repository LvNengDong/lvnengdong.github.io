Redis

  • Redis 是一款高性能的NOSQL系列的非关系型数据库。
  • Redis 是用C语言开发的一个开源的高性能键值对<K, V>数据库！
  • 比 SQL 数据库具有更快的读写速度和高并发处理能力！

2 Redis中的数据类型

Redis通过提供多种键值数据类型来适应不同场景下的存储需求，目前为止Redis支持的键值数据类型如下：

Redis在内存中以键值对的形式存储数据。其中所有键（key）都是String类型，而值（value）有5种不同的数据结构，我们常说的Redis的数据类型其实是指Redis键值对中值（value）的数据类型。

  1. 字符串类型 String
  2. 哈希类型 hash ： Map格式
  3. 列表类型 list ： LinkedList格式，支持重复元素
  4. 集合类型 set ： LinkedList格式，不允许重复元素
  5. 有序集合类型 sortedset ： LinkedList格式，不允许重复元素，且元素有序。

3 Redis命令操作

3.1 字符串类型（String）

  1. 存储：set key value

  2. 获取：get key

  3. 删除：del key

3.2 哈希类型（Hash）

  1. 存储：hset key field value

  2. 获取：

    • 获取 Hash 中指定 key 的值：hget key hashKey
    • 获取 Hash 中的所有键值对：hgetall key

  3. 删除：hdel key hashKey

3.3 列表类型（List）

  1. 添加：可以添加一个元素到列表的头部（左边）或尾部（右边）

    • lpush key value：将元素加入列表左边
    • rpush key value：将元素加入列表右边

    List和String的区别是：当key相同时存储元素，String类型是覆盖，而List是追加

  2. 获取：lrange key start end，根据范围获取。0 -1表示获取全部元素。

    • 下标从 0 开始取，左右都是闭区间，也就是说 lrange mylist 0 0 也能取到第一个元素
    • 没有rrange这个命令

  3. 删除：

    • lpop key：删除列表最左边的元素，并将元素返回
    • rpop key：删除列表最右边的元素，并将元素返回

3.4 集合类型（Set）

不允许重复元素

  1. 存储：sadd key value

  2. 获取集合中的所有元素：smembers key

  3. 删除集合中的某个元素：srem key value

3.5 有序集合类型（SortedSet）

不允许重复元素，且元素有序，每个元素都会关联一个double类型的分数（score），Redis 通过分数（score）来为集合中的成员从小到大排序。

  1. 存储：zadd key score value

  2. 获取：

    • zrange key start end：按分数从小到大的顺序获取value
    • zrange key start end withscores：按分数从小到大的顺序获取value和score

  3. 删除：zrem key value

3.6 通用命令

  1. 查询所有的键：keys *
  2. 获取指定键key对应的值value的数据类型：type key
  3. 删除指定的键值对：del key

4 持久化

Redis 是一个内存数据库，当Redis服务器重启，或者服务器电脑重启后数据就会丢失。为了解决这个问题，我们可以将Redis内存中的数据持久化到硬盘的文件中。

Redis 中的持久化机制（2种）

  1. RDB
  2. AOF

RDB

RDB：默认方式，无需进行配置，默认就使用这种机制。

AOF：通过记录日志的方式持久化 Redis 缓存中的数据。

    详细参数：

    • appendfsync always ： 每一次操作都进行持久化

5 Redis的应用场景

  • 缓存（数据查询、短连接、新闻内容、商品内容等等）
  • 聊天室的在线好友列表
  • 任务队列。（秒杀、抢购、12306等等）
  • 应用排行榜
  • 网站访问统计
  • 数据过期处理（可以精确到毫秒）
  • 分布式集群架构中的session分离
