HTTP协议

Posted on 2020-11-03 In JavaWeb

1、相关概念

  • 什么是协议？

协议是指双方或多方相互约定好，大家都需要遵循的规则。

  • HTTP：Hyper Text Transfer Protocol 超文本传输协议

所谓HTTP协议，就是指：两台或多台计算机通信时，传输数据时需要遵守的规则。

HTTP协议中的数据又叫做报文。

  • HTTP协议的特点：

    1. HTTP协议是基于TCP/IP的高级协议
    2. 默认端口号：80
    3. 基于请求/响应模型：一次请求对应一次响应。
      • 客户端浏览器访问服务器端的网页，假设一个页面上有很多内容，如图片、CSS样式、HTML静态资源、JS样式等。这些内容不是一次请求就全部响应加载出来的，而是多次请求多次响应的过程。
    4. 无状态的：
      • 每次请求之间相互独立，不能交互数据。

  • HTTP协议的历史版本

    1.0 ：每次请求都会建立新的连接。

    • 每次请求得到响应，获取资源后，都会将连接释放，下次请求会建立新的连接，周而复始。

    1.1 ：复用连接

    • 第一次请求得到响应后，在计时器未超时之前，保持与该网页服务器的连接，继续访问无需重新建立连接。

2、请求

  • 客户端给服务器发送数据叫请求。
  • 服务器给客户端回传数据叫做响应。

  • HTTP有7种请求方式，常用的有2种：

    GET：

    1. 请求参数在请求行中,位置在url之后【请求参数是请求消息的一部分，比如登录界面的账号、密码信息等】
    2. 请求的URL长度是有限制的【因为把请求参数都算作url长度】
    3. 安全性较差。

    POST：

    1. 请求参数被封装在请求体中
    2. 请求的URL长度是没有限制的。
    3. 相对安全。

3.2、常见的响应状态码

响应状态码：服务器告诉客户端本次请求和响应的状态。

  • 状态码都是3位数字

1、状态码分类（5大类）：

  • 1xx：

    • 服务器接收到了客户端的请求，但是没有接收完成，等待一段时间之后，发送1xx状态码给客户端。【客户端发送的请求消息不完整或者在传输过程中部分数据丢失，导致服务器继续等待后续的请求命令，如果等待时间大于超时时间，就会给客户端发送1xx状态码表示请求数据不完整】

  • 2xx：

    • 客户端请求消息和服务器端响应消息都成功时返回2xx状态码表示成功。
    • 这个状态码普通用户是看不到的，只有使用抓包工具才能看到.
    • 代表：200

  • 3xx

    • 重定向
    • 代表：302（重定向）、304（访问缓存）

  • 4xx

    • 客户端发生了错误
    • 代表：
      • 404：请求路径没有对应的资源
      • 405：请求方式没有对应的doXxx()方法。
        • 如：用户以post()的方式提交一张表单，但是服务器中没有doPost()方法就会出现405错误

  • 5xx

    • 服务器端发生了错误
    • 代表：
      • 500：服务器内部出现异常。
