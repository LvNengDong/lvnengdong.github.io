1、SpringBoot简介

1.1、为什么要学习SpringBoot

因为Spring框架存在一些缺点，而SpringBoot框架在一定程度上很好的解决了这些缺点，所以我们需要学习SpringBoot。

1.1.1、Spring的缺点

  1. 虽然Spring的开发代码是轻量级的，但它的配置代码却是重量级的。可以分为3个阶段：

    1. XML配置。

    2. 注解配置

    3. 基于Java类的配置

  2. 依赖管理复杂。在搭建Spring项目时，需要分析：

    1） 要导入哪些库的坐标，以及与直接导入的库有依赖关系的库的坐标

    2） 库与库之间的版本是否产生冲突

1.1.2、SpringBoot解决Spring的缺点

SpringBoot可以在很大程度上解决Spring的这2个缺点！Spring Boot 推崇约定大于配置的方式以便于我们能够尽可能快速的启动并运行程序。

  1. SpringBoot提供了约定成俗的配置文件信息，当然根据需求也可以自定义配置信息。
  2. SpringBoot框架实现了对绝大多数框架的依赖管理和依赖版本管理。

1.2、什么是SpringBoot

SpringBoot是Spring项目中的一个子工程，用一些固定的方式来构建生产级别的spring应用。Spring Boot 推崇约定大于配置的方式以便于你能够尽可能快速的启动并运行程序。其实人们把Spring Boot 称为搭建程序的脚手架。其最主要作用就是帮我们快速的构建庞大的Spring项目，并且尽可能的减少一切XML配置，做到开箱即用，迅速上手，让我们关注与业务而非配置。

Spring Boot 简化了基于Spring的应用开发，只需要“run”就能创建一个独立的、生产级别的Spring应用。Spring Boot为Spring平台及第三方库提供开箱即用的设置（提供默认设置，存放默认配置的包就是启动器），这样我们就可以简单的开始。多数Spring Boot应用只需要很少的Spring配置。

1.3、SpringBoot的核心功能

1.3.1、起步依赖

起步依赖就是将具备某种功能的坐标打包到一起，这些依赖加在一起支持某项功能。

比如，在Web开发中，我们可能要使用到Servlet、jsp、SpringMVC等等依赖，而SpringBoot就会把我们可能用到的依赖的坐标整合起来，并管理依赖的版本问题，只对外提供一个新的依赖坐标spring-boot-starter-web。这样我们在使用时只需要导入spring-boot-starter-web依赖即可，而且无需管理版本问题。

@EnableAutoConfiguration

@EnableAutoConfiguration的作用就是开启SpringBoot的自动配置功能，并且根据你引入的依赖来生成对应的默认配置。

  • 自动配置的原理就是：SpringBoot根据我们所添加的起步依赖，去“猜测”你想要如何配置该Spring项目。比如我们引入了spring-boot-starter-web起步依赖，这个起步依赖中又包含了tomcat、SpringMVC和Servlet等依赖，此时SpringBoot自动配置就会提供tomcat、SpringMVC和Servlet等组件的默认配置信息。

  • 总结：SpringBoot内部对大量的第三方库或Spring内部库进行了默认配置，这些配置是否生效，取决于我们是否引入了对应库所需的依赖，如果有那么默认配置就会生效。

  • 所以，我们使用SpringBoot构建一个项目，只需要引入所需框架的依赖，配置就可以交给SpringBoot处理了。除非你不希望使用SpringBoot的默认配置，它也提供了自定义配置的入口。

3、SpringBoot原理分析

3.3、总结

SpringBoot为我们提供了默认配置，而默认配置生效的条件一般有两个：

  • 你引入了相关依赖
  • 你自己没有配置

  1. 启动依赖

    所以，我们如果不想配置，只需要引入依赖即可，而依赖版本我们也不用操心，因为只要引入了SpringBoot提供的stater（启动依赖），就会自动管理依赖及版本了。

    因此，使用SpringBoot的第一件事情，就是找启动器，SpringBoot提供了大量的默认启动器。

  2. 全局配置

    另外，SpringBoot的默认配置，都会读取默认属性，而这些属性可以通过自定义application.properties文件来进行覆盖。这样虽然使用的还是默认配置，但是配置中的值改成了我们自定义的。

    因此，使用SpringBoot的第二件事情，就是通过application.properties来覆盖默认属性值，形成自定义配置。我们需要知道SpringBoot的默认属性key。

    SpringBoot默认会从类路径下的加载application.properties或者application.yml文件。

4.1、使用Java配置

Java配置就是：通过使用java类和一些注解，代替配置文件保存配置信息。比较常用的注解有：

  • @Configuration：作用在类上，声明该类是一个配置类，用于代替xml配置文件；
  • @Bean：作用在方法上，将方法的返回值添加到Spring核心容器中，用于代替<bean>标签；
  • @value：作用在属性上，属性注入；
  • @PropertySource：作用在类上，用来指定外部属性文件中的路径。

4.3、更简单的注入

事实上，如果一段属性（配置信息）只被一个Bean需要，我们无需将先其注入到一个类（JdbcProperties）中。而是直接在需要的地方使用@ConfigurationProperties注解声明即可。

我们直接把@ConfigurationProperties(prefix = "jdbc")声明在需要使用该段属性的@Bean的方法上，然后SpringBoot就会自动调用这个Bean（此处是DataSource）的set方法，然后完成注入。

  • 使用的前提是：该类必须有对应属性的set方法！
