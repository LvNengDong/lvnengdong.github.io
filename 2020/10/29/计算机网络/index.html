计算机网络体系结构

计算机网络概念

将多个计算机系统通过通信设备连接起来，由软件实现资源共享和信息传递的系统。

计算机网络的组成

硬件 + 软件 + 协议

  1. 硬件：
    • 主机
    • 通信链路（双绞线、光纤等）
    • 交换设备（路由器、交换机等）
    • ……
  2. 软件：实现资源共享的软件。
  3. 协议：规定了通过网络传输数据时需要遵循的规范。
    • 是两个(或多个)对等实体间通信的规则的集合，不对等的实体间是没有协议的。

计算机网络的功能

数据通信 + 资源共享 + 分布式处理 + 提高可靠性 + 负载均衡

  1. 数据通信
  2. 资源共享
  3. 分布式处理：
    • 当计算机网络中的某个计算机负荷过重时，可以将其处理的任务分配给其它计算机系统，从而提高整个系统的利用率。
  4. 提高可靠性：
    • 整个计算机网络系统就是一个大的集群，计算机网络中的各台计算机可以互为替代机。
  5. 负载均衡
    • 将工作任务均衡的分配给计算机网络中的各台计算机。

计算机网络的分类

  1. WAN + MAN + LAN + PAN
  2. 广播式网络 + 点对点网络
  3. 有线网络 + 无线网络

按分布范围分类

  1. 广域网（WAN）
  2. 城域网（MAN）
  3. 局域网（LAN）
  4. 个人区域网（PAN）

按传输技术分类

  1. 广播式网络
  2. 点对点网络

按传输介质分类

  1. 有线网络
  2. 无线网络

协议数据单元(PDU)

  • PDU（Protocol Data Unit）：协议数据单元
  • PCI（Protocol Control Information）：协议控制信息
  • SDU（Service Data Unit）：服务数据单元

对等层之间传递数据的单位称为该层的PDU。

在实际的网络中，每层的PDU都有一个别名，分别是：

层 PDU的别名
物理层 比特
数据链路层 帧
网络层 分组(/IP数据报)
传输层 报文段(TCP)/用户数据报(UDP)

服务

服务是垂直的，是指：下层为紧邻的上层提供的功能调用。

服务原语

上层要使用下层所提供的服务，必须和下层进行交互，这些交互的命令在OSI中被称为服务原语。OSI将服务原语划分为4类：

  1. 请求(Request)：服务用户–>>服务提供者，请求完成某项工作。
  2. 指示(Indication)：服务提供者–>>服务用户，指示用户做某事。
  3. 响应(Response)：服务用户–>>服务提供者，作为对指示的响应。
  4. 证实(Conformation)：服务提供者–>>服务用户，作为对请求的证实。

服务的分类

  1. 面向连接服务&无连接服务
  2. 可靠服务&不可靠服务
  3. 有应答服务&无应答服务

1、面向连接服务&无连接服务

  • 面向连接服务：建立连接 -->>数据传输 -->> 连接释放
  • 无连接服务：尽最大努力交付，是不可靠的服务
面向连接服务 无连接服务
1. 通信前是否建立连接 是 否
2. 通信前是否分配相应的资源（如内存空间） 是 否
3. 传输结束后是否释放连接 是 否
4. 常见的例子 TCP IP/UDP

2、可靠服务&不可靠服务

  • 可靠服务：网络具有纠错、检错、应答机制，能保证数据正确、可靠的传送到目的地。

  • 不可靠服务：网络只是尽量正确、可靠的传送，但不能保证数据正确、完整地传送到目的地。

    对于不可靠服务，其数据的正确性只能靠用户来保障。

3、有应答服务&无应答服务

  • 有应答服务：接收方在收到数据后向发送方给出相应的应答。
  • 无应答服务：接收方收到数据后不自动给出应答。若需要应答，则交给高层实现。

数据链路层的功能

提供点到点(不同IP地址之间)的通信

  1. 为网络层提供服务。
  2. 成帧：将网络层传递的IP数据报封装成帧。(为上层提供服务)
  3. 差错控制
  4. 流量控制
  5. 链路管理

为网络层提供服务

对网络层而言，数据链路层的功能就是将源机器中来自网络层的数据传送到目标机器的网络层中去。

数据链路层为网络层提供的服务有：

  1. 无确认的无连接服务
  2. 有确认的无连接服务
  3. 有确认的面向连接服务

链路管理

数据链路层连接的建立、维持和释放过程就称为链路管理。

差错控制

差错控制是通信的2个节点之间约定好的特定的检错规则。如奇偶校验码，接收方根据约定好的规则去检测接收到的数据是否正确。如果出了差错，能纠错就纠错，不能纠错就丢弃，确保向上层提交的数据都是无误的。

流量控制

在2个节点之间传送数据时，由于2个节点的性能不同，可能节点A的发送速度远大于B节点的接收速度，如果不加以控制，那么节点B就会丢弃很多来不及接收的数据。流量控制就是想办法协调2个节点的速度，使A节点的发送速度与B节点的接收速度达到一个平衡状态。

异构网络互联

所谓网络的互联就是指：将2个以上的计算机网络，通过一定的方法，用一种或多种通信处理设备（即中间设备）连接起来，以构成更大的网络系统。中间设备又称为中间系统或中继系统。

  • 根据中继系统所在的层次，有以下4种不同的中继系统：

    1. 物理层中的中继系统：中继器、集线器
    2. 数据链路层中的中继系统：网桥、交换机
    3. 网络层中的中继系统：路由器
    4. 网络层以上的中继系统：网关

    ​ 要实现网络的互联，每层的中继系统都必不可少。比如，我们在计算机A的浏览器上通过网关访问X资源，首先要把我们的请求信息由上到下一层层封装，最后封装为比特流，通过电信号的形式一层层的传递给存放A资源的计算机B，计算机B再把接收到的比特流逐级解封装为请求信息，作出响应。

    ​ 这一过程中，我们在每一层都可能使用了中继系统。

路由与转发

路由器主要完成2个功能：

  1. 路由选择：
  2. 分组转发：路由器根据转发表将用户的IP数据报从合适的端口转发出去。

分组转发

路由器仅根据目的主机所在的网络号来转发分组（IP数据报），从而减少了路由表所占的存储空间。

拥塞控制

数据传输一般要经过多个节点，如果某些节点接收的数据量特别大（相较于该节点接收数据的能力而言，传递的数据贼多，该节点来不及接收），那么就说网络处于拥塞状态，采取一定的方法缓解这种状况就叫拥塞控制。

IPv4地址

  • 连接在Internet中的每一台主机（或路由器）都分配一个32位的全球唯一的标识符，即IP地址。

  • IP地址可分为5类：ABCDE

  • IP地址 = {<网络号>, <主机号>}

    其中，网络号标志主机(或路由器)连接到的网络，一个网络号在因特网范围内必须是唯一的；

    主机号标志该主机(或路由器)，一个主机号在网络号所指明的网络范围内是唯一的。

  1. IP地址分为5类，这5类分别对应着5种网络号的开头
分类 网络号中对应的标示 网络号在IP地址中占的位数

C 110 24
D 1110 -
E 1111 -

  1. 在各类IP地址中，有一些IP地址表示特殊用途，不可以当做某台主机的的IP地址。

    • 主机号全为0表示本网络本身
    • 主机号全为1表示本网络的广播地址

    • 32为全为0表示本网络上的本主机。
    • 32位全为1表示整个网络的广播地址。

TCP协议的三次握手&四次挥手

TCP连接的建立

TCP连接的建立要经过3个步骤，通常称为“三次握手”。

  • step1：客户机的TCP首先向服务器的TCP发送一个连接请求报文段，这个特殊的报文段中不包含应用层数据。
  • step2：服务器的TCP接收到连接请求报文段后，如果同意建立连接，就向客户机发回确认，并为该TCP连接分配TCP缓存和变量。
  • step3：当客户机收到确认报文段后，还要想服务器给出确认，并且也要给连接分配缓存和变量。该报文段可以携带数据，如果不携带数据则不消耗序列号。

在成功进行了以上三步后，TCP连接就建立了，接下来就可以传递应用层数据了。

TCP连接的释放

参与TCP连接的两个进程中的任何一个都能终止该连接，TCP连接的释放过程通常称为“四次挥手”。

  • step1：客户机打算关闭连接，就向服务器TCP发送一个连接释放报文段，并停止再发送数据，主动关闭TCP连接。
  • step2：服务器收到连接释放报文段后发出确认。
  • step3：若服务器已经没有要向客户机发送的数据，就通知TCP释放连接，发出连接释放报文段。
  • step4：客户机收到连接释放报文段后，向服务器发出确认报文段。
