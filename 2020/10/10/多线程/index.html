多线程

Posted on 2020-10-10 Edited on 2020-12-03 In JavaSE

1、并发 vs. 并行

  • 并发：指两个或多个事件在同一个时间段内发生。
  • 并行：指两个或多个事件在同一时刻发生。（同时发生）

2、线程 vs. 进程

  1. 进程：
    • 是指一个内存中运行的应用程序，每个进程都有一个独立的内存空间；
    • 一个应用程序可以同时运行多个进程；【即一个程序占多个CPU】
    • 进程也是程序的一次执行过程，是系统运行的基本单位；
    • 系统运行一个程序即是一个进程从创建、运行、到消亡的过程。
  2. 线程：
    • 线程是进程中的一个执行单元，负责当前进程中程序的执行；
    • 一个进程中至少有一个线程；
    • 一个进程中是可以有多个线程的，这样的程序被称为多线程程序。

​ 一个程序从创建到死亡的过程称为一次进程，所以说进程就是程序的运行，一个程序至少有一个进程。一般程序都是一个进程，也有些程序如LOL就有2个进程，一个进程负责显示游戏页面，另一个进程负责显示个人主页。

​ 进程在某种意义上等同于程序，那么线程又是什么呢？在上图的谷歌浏览器中，我们看一看到运行这一进程我们使用了4个线程，这4个线程或者是并行或者是并发运行，可以充分利用计算机的性能。再举一个简单的例子就是IDM下载器，我们使用了多线程下载来提高下载速度，每个线程只负责一部分的下载任务。 多线程是可选的，因为我们执行一个进程最少情况下一个线程也能完成，但是为了提高效率、防止阻塞等原因，我们往往使用多线程来完成。

​ 并不一定说一个程序运行时使用了几个核这个程序有几个线程，因为有可能在某一个核中该程序又开辟了多个线程，只能说一个程序使用了N个核，那么这个程序的线程数必定大于等于N。

​ 其实，多线程程序并不能够提高程序的运行速度，但能够提高程序运行效率，让CPU的利用率更高。

线程调度

  1. 分时调度
    • 所有线程轮流使用CPU的使用权，平均分配每个线程占用CPU的时间。
  2. 抢占式调度
    • 优先让优先级高的线程使用CPU。如果线程的优先级相同，那么会随机选择一个使用CPU。

3、Java中创建线程的2种方法

3.1、通过Thread类创建线程

3.1.1、Thread介绍

  • Java使用java.lang.Thread类代表线程。
  • 所有的线程对象必须是Thread类或其子类的实例对象。
  • 每个线程的作用是完成一定的任务[实际上就是执行一段程序流，即顺序执行的代码]。Java使用线程执行体来代表这段代码。

3.1.2、Thread类中的构造方法

ID 方法 说明
1 Thread() 创建一个新的线程对象[，名字为系统默认]
2 Thread(String name) 分配一个指定名字的新的线程对象
3 Thread(Runnable target) 分配一个带有指定目标的新的线程对象
4 Thread(Runnable target, String name) 分配一个带有指定目标的新的线程对象并指定名字

  • Runnable target是一个接口，实际使用时接收的是该接口的实现类。

    Runnable实现类里包含了run()方法作为线程执行体，而实际的线程对象依然是Thread实例，只是该Thread线程负责执行其target对象(即Runnable接口的实现类)中的run()方法。

3.1.3、Thread类中的成员方法

ID 方法 返回值 说明
1 getName() String 获取当前线程的名称
2 start() void 使当前线程开始执行，即JVM虚拟机开始调用此线程的run()方法
3 run() void 此线程要执行的任务代码
4 sleep(long millis)[static] void 使当前正在执行的线程暂停x毫秒
5 currentThread()[static] Thread 返回当前正在执行的线程对象的引用

3.1.4、具体步骤

Java中通过继承Thread类来创建并启动多线程的步骤如下：

  1. 自定义Thread类的子类，重写类中的run()方法。
    • run()方法的方法体就是该线程需要执行的任务，因此把run()方法叫做线程执行体。
  2. 创建Thread子类的实例，即创建了线程对象。
  3. 调用线程对象的start()方法来启动该线程。

3.3、Thread和Runnable的区别与联系

实际上所有的多线程代码都是通过Thread类的start()方法来运行的。因此，不管是继承Thread类还是实现Runnable接口来实现多线程，最终还是通过Thread类的实例对象来控制线程的。

Thread和Runnable的区别主要是由继承和实现的特性决定的。如果一个类继承了Thread，由于单继承的局限性，不适合资源共享；但是如果实现了Runnable接口的话，则很容易实现资源的共享。

实现Runnable接口比继承Thread类所具有的的优势（★）

  1. 适合多个相同的程序代码的线程共享同一个资源。
  2. 可以避免Java中单继承的局限性。（如果A类已经继承了一个非Thread类，那么A类就不能再通过继承Thread类来实现多线程了，只能通过实现Runnable接口来实现多线程）
  3. 增加程序的健壮性、实现解耦操作，任务可被多个线程共享，任务和线程独立。
  4. 线程池只能放入实现Runnable或Callable类线程，不能直接放入继承Thread的类。

4、线程安全

4.1、线程安全

  • 如果有多个线程在同时运行，而且这些线程可能会同时运行某段代码。 如果多线程程序每次运行结果和单线程运行的结果是一样的，而且其他变量的值也和预期的是一样的，就说线程是安全的。
  • 线程安全问题都是由全局变量和静态变量引起的。 若每个线程中对全局变量、静态变量只有读操作，而无写操作，一般来说，这个全局变量是线程安全的。 若有多个线程同时执行写操作，一般都需要考虑线程同步，否则的话就可能影响线程安全。

5、线程状态

5.1、线程状态分类

线程状态 发生的条件
New 线程刚被创建，但是并未启动，还没有调用start()方法。
Runnable 线程可以在Java虚拟机中运行，但不一定正在运行，这取决于操作系统的处理器。[如在单核系统中有2个线程正在正常运行，那么任一时刻必定只有一个线程在运行而另一个线程在等待运行]
Blocked 当一个线程试图获取一个锁对象，而该锁对象被其它线程持有，则该线程进入Blocked状态，当该线程持有锁时，该线程将变成Runnable状态。
Waiting 一个线程在等待另一个线程执行唤醒动作时，该线程进入Waiting状态。进入这个状态时不能自动唤醒的，必须等待另一个线程调用notify或notifyAll方法才能唤醒。
TimedWaiting 有几个方法有超时参数，调用它们将进入TimedWaiting状态，这一状态将一直保持到超时期满或者接收到唤醒通知。 常用的带有超时参数的方法有**Thread.sleep()、Object.wait()**
Teminated 因run()方法的正常退出而死亡，或者因为没有捕获的异常终止了run()方法而死亡。

常用的唤醒方法

  1. Object.notify()
  2. Object.notifyAll()

5.2、Runnable

  • 顾名思义，Runnable状态就是一种可运行状态，可运行就是可能正在执行，也可能没有在执行。
  • 多线程的程序在没有进行到争夺同一锁对象之前都是Runnable状态，但是一旦涉及到争夺锁对象，就会分为Runnable和Blocked两种状态。

5.3、TimedWaiting

  • 进入TimedWaiting状态的一种方法是调用Thread.sleep()方法，单线程也可以调用。
  • 为了让其它线程也有机会执行，可以将Thread.sleep()的调用放入线程执行体run()之内，这样才能保证该线程执行过程中会睡眠。[睡眠是在线程已经执行了一部分的情况下进入的，我们一般会把sleep()方法写在线程执行体的前面几行代码中，使当前线程进入睡眠]
  • sleep()中指定的时间是线程不会运行的最短时间。因此，sleep()方法不能保证该线程睡眠时间到期后就立即开始执行，而是会进入Runnable状态。

TimeWaiting线程状态图

5.4、Blocked（锁阻塞）

​ 当多个线程协作时，如A，B线程，如果线程A和线程B代码中使用同一锁对象，如果线程A获取到锁，线程A进入到Runnable状态，那么线程B就进入到Blocked锁阻塞状态。

Blocked线程状态图

5.5、Waiting（无限等待）

​ 当多个线程协作时，如A、B线程，如果A线程在Runnable状态种调用了wait()方法，那么A线程就进入了Waiting状态，*同时失去了锁对象（Tip：sleep方法不会释放锁对象）*。 如果此时B线程获取到了锁对象，并在Runnable状态中调用了notify()方法，那么就会将Waiting状态的A线程唤醒。注意是唤醒，如果A线程被唤醒后获取到锁对象，那么就进入Runnable状态，如果没有获取到锁对象，则进入Blocked状态。

6、等待/唤醒机制

6.1、线程间通信

概念

​ 多个线程在处理同一资源，但是处理的动作（线程的任务）却不相同。

​ 比如，线程A是一个生产包子的线程，线程B是一个消耗包子的线程，那么包子就可以理解为同一资源，线程A和B处理包子的动作不同，一个是生产，一个是消耗，那么线程A和B之间就存在线程通信问题。

为什么要处理线程间通信

​ 多个线程并发执行时，在默认情况下CPU是随机切换线程的，当我们需要多个线程来共同完成一件任务，并且我们希望他们有规律的执行，那么多个线程之间就需要一些协调通信，以此来帮助我们达到多线程共同操作一份数据。

​ 比如，线程A是生产包子的线程，线程B是消耗包子的线程，那么我们希望线程A每生产100个包子就通知线程B开始吃包子，线程B吃完包子后就通知线程A再开始生产包子，这时我们就必须让线程之间进行同行来达到这一目的。 对于线程A，设置一个阀值，当包子数量达到阀值后线程A进入睡眠状态；此时线程B开始吃包子，当包子数量为0时线程B进入睡眠状态，并唤醒线程A让线程A继续生产包子。

如何保证线程间同行有效利用资源

​ 多个线程在处理统一资源，并且任务不相同时，需要线程通信来帮助解决多个线程对于同一变量的操作可能发生的冲突。我们一般采用的手段是——等待/唤醒机制。

6.2、wait/notify机制

​ 等待/唤醒机制就是用于解决线程间通信的问题的，常试用的方法有3个，分别是：

方法 说明
wait() 让当前线程进入等待队列中；
notify() 唤醒所通知对象等待队列中的一个线程，重新进入到调度队列中；
notifyAll() 唤醒所通知对象等待队列中的全部线程。

调用wait()和notify()方法需要注意的细节

  1. wait()和notify()方法必须要由同一个对象调用。只有同一个对象的notify()方法才能唤醒由该锁对象调用wait()方法后的线程。
  2. wait()和notify()方法是属于Object类的方法。因此，锁对象可以是任意对象。
  3. wait()和notify()方法必须要在同步代码块或同步函数中使用。因为，必须通过锁对象调用这2个方法。

7、线程池

7.1、概念

  • 线程池：一个可以容纳多个线程对象的容器，其中的线程可以反复使用，省去了频繁创建线程对象的操作，无需反复创建线程对象而消耗过多的资源。

使用线程池的好处

  1. 降低资源消耗。

    ​ 减少了创建和销毁线程的次数，每个线程都可以被反复利用。

  2. 提高响应速度。

    ​ 当任务到达时，任务可以不需要等待线程的创建就能立即执行。[直接从线程池中调用线程对象]

  3. 提高线程的可管理性。可以根据系统的承受能力，调整线程池中工作线程的数据，防止因为消耗过多的内存而导致服务器奔溃。(每个线程大约需要1MB的内存，线程开的越多，消耗的内存也就越大，最后死机)

    线程池会事先设置好该池内会有多少个线程，假设某一时刻线程池中的线程全部处于运行状态，这时如果还有新的任务需要申请新的线程，该任务不会创建新的线程，而是进入等待状态，直到线程池中的某个线程处于空闲状态，再调用该线程去执行任务。

7.2、线程池的使用

Java中线程池的顶级接口是java.util.concurrent.Executor，但是严格意义上讲Executor并不是一个线程池，而是一个线程池执行工具。真正的线程池接口是java.util.concurrent.ExecutorService。

要配置一个线程池是比较复杂的，因此官方提供了一个工具类java.util.concurrent.Executors线程工厂类，里面提供了一些静态工厂，生成一些常用的线程池。

  • 多个线程在处理同一资源，但是处理的动作（线程的任务）并不相同，那么这多个线程间就必须保持通信来维持资源的正确性。
