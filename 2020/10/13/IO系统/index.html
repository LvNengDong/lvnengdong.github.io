1、File类

1.1、File类概述

  • java.io.File类是文件和目录路径名的抽象表示形式。 主要用于文件和目录的创建、查找和删除等操作。

1.2、File类中的方法

1.2.1、构造方法

ID 方法 说明
1 File(File parent, String child) 根据 parent 抽象路径名和 child 路径名字符串创建一个新 File 实例。
2 File(String parent, String child) 根据 parent 路径名字符串和 child 路径名字符串创建一个新 File 实例
3 File(String pathname) 通过将给定路径名字符串转换为抽象路径名来创建一个新 File 实例。
4 File(URL uri) 通过将给定的 file: URI 转换为一个抽象路径名来创建一个新的 File 实例

  • File 类没有无参构造方法。也就是说创建 File 实例必须指定文件/目录要保存的位置。
  • File实例可以是一个文件，也可以是一个目录。
  • 创建File实例并不等价于创建了文件/文件夹。 创建 File 实例只是在内存中开辟了一块空间保存这个实例对象，并不会在硬盘上创建一个真实的文件或文件夹，只有调用这个实例对象的 mkdir()、mkdirs() 或 createNewFile() 方法时才会在硬盘上创建出真正的文件/目录。

1.2.3、成员方法

ID 方法 返回值 说明
获取功能：
1 getAbsolutePath() String 返回此File的绝对路径字符串
2 getPath() String 将此File转换为路径名字符串
3 getName() String 返回此File表示的文件或目录名称。
4 length() long 返回此File表示的文件的长度[字节]。
判断功能：
6 exists() boolean 判断此File表示的文件或目录是否存在
7 isDirectory() boolean 判断此File表示的是否为目录。
8 isFile() boolean 判断此File表示的是否为文件
创建/删除功能
9 createNewFile() boolean 当且仅当具有该名称的文件尚不存在时，创建一个新的文件。
10 delete() boolean 删除此File表示的文件或目录。【如果此File表示目录，则目录必须为空才能被删除】
11 mkdir() boolean 创建此File表示的目录
12 mkdirs() boolean 创建此File表示的目录，包括任何必需但不存在的父目录。
目录的遍历
13 list() String[] 返回一个String数组，返回该File目录中的所有子文件或目录。
14 listFiles() File[] 返回一个File数组，表示该File目录中的所有子文件或目录。
  • 调用list或listFiles方法遍历File对象，此时File对象必须是目录，且真实存在，否则会返回空指针异常，无法进行遍历。

mkdir() 、mkdirs() 和 createNewFile() 的区别

  • mkdir() / mkdirs() 方法用于创建目录，createNewFile() 方法用于创建文件。

    也就是说，在使用mkdir() / mkdirs() 方法时，即使开发人员指定目录名为 aaa.txt ，它也不会创建出一个 aaa.txt 文件，而只是会创建出一个名为 aaa.txt 的目录。

  • mkdirs() 方法可用于一次性创建多级目录，而 mkdir() 和 createNewFile() 方法每次只能创建单级目录。

    也就是说，如果指定的文件路径是一个多级目录，且这个多级目录中间的部分目录不存在，则创建文件/目录的方法就不会执行成功。

    不同的是 mkdir() 方法即使执行不成功也不会报错，而 createNewFile() 方法会直接报错。报错信息如下：

3、字节流

3.0、一切皆为字节流

一切文件数据（文本、图片、视频等）在存储时，都是以二进制数字的形式保存的，都是一个一个的字节，传输时同样如此。 所以，字节流可以传输任意文件数据。 在操作流时，我们要时刻明确，无论使用什么样的流对象，底层传输的始终是二进制数据。

3.1、OutputStream

3.1.1、概述

  • java.io.OutputStream是抽象类 表示字节输出流的所有类的超类。
  • 将指定的字节信息写出到目的地。
  • 它定义了字节输出流的基本的公共的方法。

3.1.2 构造方法

  • OutputStream 类中没有专门定义构造方法，所以 OutputStream 类使用 JVM 提供的默认无参构造器！！！

3.1.3 成员方法

ID 方法 返回值 说明
1 close() void 关闭此输出流并释放与此流相关的任何系统资源
2 flush() void 刷新此输出流并强制任何缓冲的输出字节流被写出。
3 write(byte[] b) void 将b数组的内容写入此输出流。
4 write(byte[] b, int off, int len) void 从指定的字节数组写入len字节，从偏移量off开始输出到此输出流
5 write()[abstract] void 将该字节输出流写出到目的地。
  • 当完成流的操作时，必须调用close()方法，释放系统资源。

3.2、FileOutputStream

3.2.1、概述

  • FileOutputStream类是文件输出流，用于将数据写出到文件。

3.2.2、构造方法

ID 方法 说明
1 FileOutputStream(File file) 创建文件输出流，并指定写出的目标文件。
2 FileOutputStream(String name) 创建文件输出流，并指定写出的目标地址。
3 FileOutputStream(File file, boolean append) 数据可追加续写
4 FileOutputStream(String name, boolean append) 数据可追加续写
  • 在创建字节输出流对象 FileOutputStream 时，必须指定该流对象要输出到的目的地（可以是一个 File 对象，也可以是一个 String 类型的路径）。 在该路径下，如果没有这个文件，会自动创建该文件。

3.3、InputStream

  3. 它定义了字节输入流的基本的公共的方法。
  • 当完成了流的操作时，必须调用close()方法，释放系统资源

3.3.2、构造方法

  • InputStream 类中没有专门定义构造方法，所以 InputStream类使用 JVM 提供的默认无参构造器！！！

ID 方法 返回值 说明

2 read()[abstract] int 从输入流中读取数据的下一个字节
3 read(byte[] b) int 从输入流中读取一些字节数，并把它们存储到字节数组b中
read(byte b[], int off, int len)

4、字符流

4.1、Reader

4.1.1、概述

  1. java.io.Reader是抽象类，是所有读取字符流类的超类。可以读取字符信息到内存中。
  2. 它定义了字符输入流的共性方法。

  1. 字符流只能操作文本文件，不能操作图片、视频等非文本文件。

    当使用字节流读取文本文件时，可能会出现一个小问题。那就是遇到中文字符时，可能不会显示完整的字符。因为一个中文字符可能占用多个字节存储，而字节流读取是以单个字节为单位的，所以可能会出现一个中文字符占了3个字节，使用字节流读取时则把这3个字节分开来展示，这样就会出现乱码现象。 所以Java提供一些字符流类，以字符为单位读写数据，专门用于处理文本文件。

4.1.2、构造方法

  • Reader 类中没有专门定义构造方法，所以 Reader 类使用 JVM 提供的默认无参构造器！！！

4.1.3、成员方法

ID 方法 返回值 说明
1 close() void
2 read() int 从输入流中读取一个字符
3 read(char[] cbuf) int 从输入流中读取一些字符，并将它们存储到字符数组cbuf中。

4.2、FileReader

4.2.1、概述

  1. 构造时使用系统默认的字符编码和默认字节缓冲区。
    • 字符编码：字节与字符的对应规则。Windows系统的中文编码默认是GBK编码表；IDEA中默认UTF-8
    • 字节缓冲区：一个字节数组，用来临时存储字节数据。

4.2.2、构造方法

ID 构造方法 说明
1 FileReader(File file)
2 FileReader(String fileName)
  • 当你创建一个FileReader流对象时，必须传入一个文件路径。

4.2.3、成员方法

ID 方法 返回值 说明
读取字符数据
read() int 每次读取一个字符的数据，提升为int型，读到文件末尾返回-1
read(char[] cbuf) int 每次读取cbuf.length个字符到数组中，返回读取到的字符个数，读取到末尾时返回-1.

4.3、Writer

4.3.1、概述

  1. java.io.Writer是一个抽象类，是所有写出字符流的超类。负责将指定的字符信息写出到目的地。
  2. 定义了字符输出流的基本的共性的方法。

4.3.2、构造方法

  • Writer 类中没有专门定义构造方法，所以 Writer 类使用 JVM 提供的默认无参构造器！！！

4.3.3、成员方法

ID 方法 返回值 说明
1 write(int c) void 写入单个字符
2 write(char[] cbuf) void 写入字符数组
3 write(char[] cbuf, int off, int len) void 写入字符数组的某一部分
4 write(String str) void 写入字符串
5 write(String str, int off, int len) void 写入字符串的某一部分
6 flush() void 刷新该流的缓冲
7 close() void 关闭此流，关闭前先刷新一次

4.4、FileWriter类

4.4.1、概述

  1. java.io.FileWriter类是写出字符到文件的类，构造时使用系统默认的字符编码和默认字节缓冲区。

  • FileWriter 类中全部都是构造方法，普通方法直接沿用父类 Writer 中的方法

4.4.2、构造方法

ID 方法 说明
1 FileWriter(File file)
2 FileWriter(String fileName)
3 FileWriter(File file, boolean append)
4 FileWriter(String fileName, boolean append)

4.4.3、成员方法

ID 方法 返回值 说明
写出数据
write(int b) 每次可以写出一个字符数据
write(char[] cbuf)
write(char[] cbuf, int off, int len)
write(String str)
write(String str, int off, int len)
关闭和刷新
flush() 刷新缓冲区，流对象可以继续使用
close() 先刷新缓冲区，然后通知系统释放资源

4.4.4、关闭和刷新

​ 因为FileWriter类内置缓冲区的原因，写出数据时首先会被写出到缓冲区中，如果不进行刷新的话，是无法写入到文件中的。

  • flush()：刷新缓冲区，流对象可以继续使用；
  • close()：先刷新缓冲区，然后通知系统释放资源。流对象不可以再被使用了。

5、Properties类

5.1、包

  1. Properties类表示一个持久的属性集。
  2. 它使用键值对结构存储数据，每个键及其对应的值都是一个字符串。

5.2、构造方法

ID 构造方法 说明
1 Properties() 创建一个空的属性列表
2 Properties(Properties default) 创建一个带有指定默认值的空属性列表

5.3、成员方法

ID 方法 返回值 说明
1 setProperties(String key, String value) Object 保存一对属性
2 getProperties(String key) String
3 stringPropertyNames() Set<String> 获取所有键的名称的集合
4 load(InputStream inStream) void 从字节输入流中获取键值对

与流相关的方法

  • public void load(InputStream inputStream)：从字节输入流中读取键值对。

    ​ 字节数据流中的数据是从外部存储设备读取到内存中的，而Properties类又可以读取InputStream中的数据，这样Properties就可以间接的加载外部存储设备中的文本数据了。

    ​ 但是要求文本数据的格式符合键值对的形式，一般使用.properties或.yml格式。
