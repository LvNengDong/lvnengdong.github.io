JVM——本地方法库

Posted on 2021-05-20 Edited on 2021-08-03 In JVM

本地方法接口

什么是本地方法？

  • 使用 native 关键字修饰的方法。
  • 该方法由非 Java 语言实现的，比如 C 或 C++。
  • 在定义一个 Native Method 时，并不提供实现体（有点像定义接口中的抽象方法），因为其具体实现是由非 Java 语言在外面实现的。
  • 标识符 native 可以与所有其它的 java 标识符连用，但 abstract 除外。
  • 一个 Native Method 就是一个Java调用非Java代码的接口。
  • 这个特征并非 Java 特有的，很多其它的编程语言都有这一机制，比如在 C++ 中，就可以用 extern "C" 告知 C++ 编译器去调用一个 C 的函数。
  • 本地接口的作用是融合不同的编程语言为 Java 所用，它的初衷是融合 C/C++ 程序。

为什么要使用 Native Method ？

Java 语言使用起来非常方便，然而有些层次的任务用 Java 实现起来不容易，或者我们对程序的效率很在意时，问题就来了。

  1. 与Java环境外交互

有时Java应用需要与Java外面的环境交互，这是本地方法存在的主要原因。有时Java需要与一些底层系统，如操作系统或某些硬件交换信息，本地方法正是这样的一种交流机制，它为我们提供了一个非常简洁的接口，类似于一个黑盒，我们无需去了解Java应用之外的繁琐的细节。

  1. 与操作系统交互
  • JVM 虚拟机是一个虚拟机系统，支持 Java 语言本身和运行时库，它是Java程序赖以生存的平台，它由一个解释器（解释字节码）和一些连接到本地代码的库组成。
  • 然而不管怎样，它毕竟不是一个完整的系统，它经常需要依赖一些底层系统的支持。这些底层系统常常是强大的操作系统。
  • 通过使用本地方法，我们得以用 Java 与底层的操作系统交互，甚至JVM的一些部分就是用C写的。
  • 还有，如果我们要使用一些 Java语言本身没有提供封装的操作系统特性时，我们也需要使用本地方法。
  1. Sun’s Java

Sun的解释器是用C实现的，这使得它能像一些普通的C一样与外部交互。JRE 大部分是用 Java 实现的，它也通过一些本地方法与外界交互。例如：java.lamg.Thread 的 setPriority() 方法是用Java实现的，但是它的具体实现调用的是该类里的本地方法setPriority0()。这个本地方法是用C实现的，并被植入JVM内部。

现状

目前本地方法的使用越来越少了，除非是与硬件有关的应用，比如通过 Java 程序驱动打印机或者 Java 系统管理生产设备，在企业级应用中已经比较少见。因为现在的异构领域间通信很发达，比如可以使用Socket通信，也可以使用Web Service等等。

Token

完成登录后，会将一些基本用户信息，如 用户名、手机号、IP地址等 返回给前端去展示。

Token 就是将这些信息包装成一个编码后的字符串，如

并且将这个字符串放入请求头 header 中。

在一些需要用户登录才能完成操作的页面，当发起请求时，每次都需要进行登录判断。

在请求头中，查看是否包含 token 字符串，

如果包含，对 token 字符串进行校验，比较该 token 与我们生成的 token 是否相同，相同则校验通过，表示目前就处于登录状态。否则不通过。

token 可以设置有效时间！

Token 的生成

Token 需要按照一定的规则来生成。这个规则可以由开发者自己决定，也可以借助一些 token 生成工具来生成。

JWT 就是一个生产 token 的工具包。

JWT

  • JWT（Json Web Token）：是为了在网络应用环境间传递声明而执行的一种基于JSON的开放标准。（就是按照一定的规则生成 Json 字符串）
  • JWT 的声明一般被用来在身份提供者和服务提供者间传递被认证的用户身份信息，以便于从资源服务器获取资源。比如用在用户登录上。
  • JWT最重要的作用就是对 token信息的防伪作用。

JWT 原理

一个JWT由三个部分组成：公共部分、私有部分、签名部分。最后由这三者组合进行 base64 编码得到 JWT。

  1. 公共部分
  • 主要是该 JWT 的相关配置参数，比如签名的加密算法、格式类型、过期时间等等。
  • Key=ATGUIGU
  1. 私有部分
  • 用户自定义的内容，真正要封装的信息。
  • 如：user { “userId”：”192321”，”name”：”张三”}
  1. 签名部分
  • SaltiP：当前服务器的 IP 地址！{linux 中配置代理服务器的ip}
  • 主要用户对 JWT 生成字符串的时候，进行加密{盐值}

最终组成：key+salt+userInfo –>>token

base64 编码，并不是加密，只是把明文信息变成了不可见的字符串。但是其实只要用一些工具就可以把 base64 编码解成明文，所以不要在JWT中放入涉及私密的信息。

在 Linux 上安装好 MongoDB 后，如果发现通过远程仍然不能访问到 MongoDB 数据库，这时主要需要检查两点：

问题

使用 Spring 的拦截器获取请求参数时，发现一个问题就是：

GET 请求使用 request.getParameterMap() 获取请求参数是可行的，

但是 POST 请求使用 request.getParameterMap() 获取请求参数是获取不到的。

原因

如果是 POST请求 contentType 设置为 application/x-www-form-urlencoded 是可以通过 request.getParameterMap() 来获取请求参数的；

application/x-www-form-urlencoded 即POST请求的请求参数是键值对类型。

如果上述条件没有满足，则相关的表单数据不会被设置进 request 的 parameter 集合中，相关的数据就需要通过 request.getInputStream() 来拦截获取。

HTTP 请求头中的 content-type 属性

什么是 Content-Type ？

HTTP 消息的请求头/响应头数据是以键值对<K, V>形式存储的。 Content-Type 就是 HTTP 请求头消息中的一个键值对的 key ，用来描述客户端浏览器向服务器发送的请求时请求数据的数据类型，其对应的 Value 是 MIME 类型的数据。

Tip：

HTTP 协议不仅是一个数据传输的工具，还能改变数据在网络中传输的类型。

具体想要数据以什么样的类型在网络中传输传递则是由前端人员提前规定好的。！！！

文件上传前置知识

使用HTTP协议上传文件时数据传输的类型

Content-Type: multipart/form-data ： 使用HTTP协议进行文件传输时数据的格式

表示将客户端提交的数据，以多段的形式进行拼接，然后以二进制流的形式发送给服务器。

  • 多段：一次可上传多条数据，每条数据之间以分隔符分开。每条属性都是二进制的字符流。

在通过HTTP协议（一般是浏览器）上传文件（如图片）时，显然简单的<K, V>值是不能描述图片对象的，所以我们在上传文件的同时要指定Content-Type: multipart/form-data属性值，这样 HTTP 协议在接收到这两个条件后，就会将客户端的上传的图片文件转化成（多段）二进制字节流数据，封装到请求体中，最后将这个HTTP请求中包含的所有信息交给到服务器处理。

文件上传的必要前提（前端）

  1. 设置表单属性：<from enctype = "multipart/form-data">
  • 如果想要 HTTP 协议能够将表单中的数据转化成多段二进制流数据的话，在上传文件的同时我们就要声明上传的文件在HTTP中的数据类型是Content-Type: multipart/form-data，在前端中的做法就是设置表单属性为：<from enctype = "multipart/form-data">
  • 在前端的表单中，<form encType=""> 中默认的 encType 是 application/x-www-form-urlencoded，这表示 <form/> 表单中的数据将被HTTP协议编码为 <K, V> 格式发送到服务器。而如果我们想通过表单进行文件上传的话，需要修改表单的属性为：<from encType=multipart/form-data>
  • 不管我们通过前端页面还是通过 Postman 等工具进行文件上传时都必须设置请求头中 Content-Type 的值为 multipart/form-data 。
  1. 请求方式必须是POST

原因：GET 请求有长度的限制，使用 GET 请求上传图片的话往往会超出请求URL 的长度限制，导致上传图片不能正常显示！！！

  1. 提供一个文件选择域<input type="file"/>

HTML <form> 标签的 enctype 属性

说明：

enctype 属性规定在发送到服务器之前HTTP协议应该如何对<form>表单中的数据进行编码。

默认地，表单数据会编码为 “application/x-www-form-urlencoded”。也就是说，在发送到服务器之前，所有字符都会进行 url 编码（空格转换为 “+” 加号，特殊符号转换为 ASCII HEX 值）。

语法：

属性值：

值 描述
application/x-www-form-urlencoded 在发送前编码所有字符（默认）
multipart/form-data 不对字符编码。在使用包含文件上传控件的表单时，必须使用该值。
text/plain 空格转换为 “+” 加号，但不对特殊字符编码。

Spring MVC中关于Content-Type类型信息的使用

在Spring MVC中，主要就是使用@RequestMapping注解来处理请求，因此首先我们来看看@RequestMapping注解的Class定义。

下面对注解提供的参数进行简单解析。

value：指定请求的 URL

method： 指定请求的method类型， 比如GET、POST、PUT、DELETE等。

consumes：指定处理请求的提交内容类型（Content-Type），例如application/json、text/html;等。

produces：指定返回的内容类型，仅当request请求头中的(Accept)类型中包含该指定类型才返回。

params：指定request中必须包含某些参数值，才让该方法处理。

headers：指定request中必须包含某些指定的header值，才能让该方法处理请求。

其中，consumes、produces可以使用Content-type中的信息对不需要的信息进行过滤，仅接受需要的数据；headers则可以使用Content-Type中的信息对请求进行过滤和判断。

什么是路由

路由就是一组映射关系，即<K, V>键值对。

K 是 Path，

V 是 通过这个 Path 要访问的路径！

  • 在后端中，V 一般是处理请求的回调函数。
  • 在前端（Vue）中：V 就是一个组件。

Vue-Router 概述

  • Vue-Router 是官方提供的用来实现 SPA 的 vue 插件
  • GitHub地址：https://github.com/vuejs/vue-router
  • 官方文档（中文）：http://router.vuejs.org/zh-cn/
  • 下载：npm install vue-router --save

  • 注册路由器就是声明使用路由器，我们创建好了 Vue-Router 实例后如果不告诉 Vue 的话 Vue 在启动项目时也不会去加载Vue-Router这个路由器实例。声明使用类似于 SpringBoot 中的 @Component 注解，只有声明使用后项目才会去加载该类中的信息。
