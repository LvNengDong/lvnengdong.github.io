Listener

Posted on 2020-04-20 Edited on 2020-11-03 In JavaWeb

1、Listener概念

监听器就是监听某个对象状态变化的组件。

常用于监听web应用中某些对象的创建、销毁、增加，修改，删除等动作的发生，然后作出相应的响应处理。

当对象的状态发生变化的时候，服务器自动调用监听器对象中的方法。

  1. 概念：Listener，监听器。Web三大组件之一。
  2. 事件监听机制
    • 事件 ：一件事情
    • 事件源 ：事件发生的地方
    • 监听器 ：一个对象
    • 注册监听：将事件、事件源、监听器绑定在一起。 当事件源上发生某个事件后，执行监听器代码

2、Listener的实现

2.1 实现步骤

  1. 自定义一个监听器类，实现ServletContextListener接口。
  2. 重写接口中的方法，完成监听的逻辑。
  3. 在配置文件web.xml中（或使用注解）配置监听对象。

2.2 使用web.xml配置文件配置

  • 在配置文件中将自定义的监听器类注册到监听器列表中去。

2.3 注解配置

  • 在自定义的监听器类上使用@WebListener注解将该类注册到监听器列表中去。

3、ServletContextListener

  • 作用：监听ServletContext对象的创建和销毁

  • 方法：

    1. void contextDestroyed(ServletContextEvent sce)
      • ServletContext对象被销毁之前会自动调用该方法
    2. void contextInitialized(ServletContextEvent sce)
      • ServletContext对象创建之后会自动调用该方法。

1. Filter原理

Filter是对客户端访问资源的过滤，符合条件放行，不符合条件不放行，并且可以对目标资源访问前后进行逻辑处理。

Filter主要用于对用户请求（request）进行预处理，也可以对服务器响应（response）进行后处理。

  • 当浏览器访问服务器的资源时，过滤器可以将请求拦截下来，完成一些特殊的功能。
  • 过滤器一般用于完成通用的操作。如，登录验证、统一编码处理、敏感字符的操作等。

2. Filter实现

2.1 实现步骤

  1. 自定义一个过滤器（Filter接口的实现类）。

  2. 重写接口中的方法，完成过滤器逻辑的编写。

  3. 设置过滤条件（拦截条件）

    • web.xml配置方式
    • 注解配置方式

2.2 过滤条件

2.2.1、通过访问路径拦截请求消息

  1. 拦截具体资源
    • /index.jsp 只有访问index.jsp时，才会执行该过滤器。
  2. 通过目录拦截
    • /user/* 访问/user目录下的所有资源时，过滤器都会被执行。
  3. 通过后缀名拦截
    • *.jsp 访问所有后缀名为.jsp的资源时，过滤器都会被执行。
  4. 拦截所有资源
    • /* 访问所有资源时，过滤器都会被执行。

2.2.2、通过访问方式拦截请求消息

  1. REQUEST 默认值[拦截浏览器直接发起的Request请求]

  2. FORWARD 只拦截来自请求转发的访问

  3. INCLUDE 只拦截包含访问资源

  4. ERROR 只拦截错误跳转资源

  5. ASYNC 只拦截异步访问资源

  6. 组合形式

2.3 实现方式

2.3.1、配置文件实现

  • 配置文件位置：Web/WEB-INF/web.xml
  • 配置<dispatcher></dispatcher>标签可以完成调度。可以设置一个或多个。

2.3.2、注解实现

设置dispatcherTypes属性【Dispatcher：调度器】

  1. REQUEST 默认值[拦截浏览器直接发起的Request请求]

  2. FORWARD 只拦截来自请求转发的访问

  3. 组合形式

3. 过滤器细节

3.1 过滤器执行流程

  1. 请求消息到达过滤器后首先执行过滤器中放行前的业务逻辑；
  2. 放行后，跳转到Servlet转去执行Servlet中的业务逻辑；
  3. Servlet给出的响应消息返回给客户端时再次经过过滤器，执行过滤器中放行后的业务逻辑。

3.2 过滤器生命周期方法

  1. init()
    • 服务器在启动时，会自动创建Filter实例，该实例调用init()方法。只执行一次。用于加载资源。
  2. doFilter()
    • 每一次请求被拦截资源时，都会执行。执行多次。
  3. destroy()
    • 服务器正常关闭时，会调用destroy()方法，Filter对象被销毁。只执行一次，用于释放资源。

3.5 过滤器链（配置多个过滤器）

3.5.1. 过滤器执行顺序：栈式原则。

​ 如果有2个过滤器，过滤器1和过滤器2。其执行顺序为：

  1. 过滤器1
  2. 过滤器2
  3. 访问资源
  4. 过滤器2
  5. 过滤器1

3.5.2. 过滤器先后顺序问题

  • 如果有2个过滤器，过滤器1和过滤器2，如何判断是过滤器1先执行还是过滤器2先执行？

1. 注解配置

  • 按照类名的字符串比较规则，值小的先执行。

2. web.xml配置

  • 在配置文件中，谁定义在前面谁先执行。
